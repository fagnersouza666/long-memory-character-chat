# Plano 1 — Setup e Execução Local

Este plano descreve passos mínimos (não destrutivos) para iniciar o projeto (UI em Streamlit e backend em FastAPI) e validar os testes do backend.

1) Preparar ambientes Python 3.12
- UI (raiz):
  - `python -m venv .venv && source .venv/bin/activate`
  - `pip install -r requirements.txt`
- Backend:
  - `cd backend`
  - `python -m venv .venv && source .venv/bin/activate`
  - `pip install -r requirements.txt`

2) Configurar variáveis de ambiente
- Backend: exportar `DATABASE_URL=postgresql://user:pass@localhost:5432/rag_enterprise` (ajuste usuário/senha/host conforme seu setup).

3) Iniciar Backend (FastAPI)
- `cd backend && source .venv/bin/activate`
- `uvicorn app.main:app --reload`
- Verificar em `http://localhost:8000/docs` se as rotas carregam.

4) Iniciar UI (Streamlit)
- Na raiz (venv da raiz ativo):
- `streamlit run app.py` (chat) ou `streamlit run rag_app.py` (RAG empresarial)
- Acessar o link exibido (ex.: `http://localhost:8501`).

5) Executar testes do backend
- `cd backend && source .venv/bin/activate && pytest -q`
- Registrar saída (esperado: testes passando ou falhas a corrigir).

6) (Opcional) Prisma/Migrações e seed
- Apenas se editar `backend/prisma/schema.prisma`:
  - `cd backend && npx prisma migrate dev`
- Opcional: popular modelos com `python setup_models.py`.

Observações
- PEP 8 + type hints. Commits no padrão Conventional Commits.
- Não executar comandos destrutivos (ex.: prune, reset, rm -rf).
